# 🧠 NLP Sentiment Analyzer — MoodMind AI

> **Understand the emotion hidden inside every review.**

A Deep Learning-powered sentiment analysis web application that classifies customer reviews as **Positive** or **Negative** using Natural Language Processing and a custom-trained Neural Network — all wrapped in a clean, interactive **Streamlit** interface.

---

## 📌 Project Overview

**MoodMind AI** is an end-to-end NLP pipeline built to analyze the sentiment of text reviews. It leverages:

- **FastText** word embeddings for rich semantic vector representations
- **spaCy** for intelligent text preprocessing (lemmatization, stopword filtering)
- A custom **Deep Learning model** (Keras/TensorFlow) trained on the IMDB dataset
- **Streamlit** for a real-time interactive web UI

## 🤖 Models Used

### 1. 🔤 FastText — Word Embedding Model
- **Library**: `fasttext-wheel`
- **Purpose**: Converts individual words into 300-dimensional dense vectors capturing semantic meaning
- **Training**: Trained on the IMDB review corpus (`review.txt`)
- **Output file**: `embeddingModel.bin` *(auto-generated when you run `app.ipynb`)*

### 2. 🧠 Custom Deep Learning Classifier — Keras/TensorFlow
- **Framework**: TensorFlow / Keras
- **Architecture**: Dense Neural Network
- **Input**: 300-dimensional mean-pooled FastText sentence vector
- **Output**: Single sigmoid neuron → probability score (0 = Negative, 1 = Positive)
- **Trained on**: IMDB Dataset (50,000 labeled movie reviews)
- **Output file**: `ReviewAnalyiser.h5` *(auto-generated when you run `app.ipynb`)*

### 3. 📝 spaCy — NLP Preprocessing
- **Model**: `en_core_web_sm`
- **Purpose**: Tokenization, lemmatization, punctuation & stopword removal
- **Smart stopword handling**: Preserves negation words (`not`, `no`, `never`) to maintain sentiment context

---

## ⚙️ How the Pipeline Works

```
Raw Review Text
      │
      ▼
[ spaCy Preprocessing ]
  • Lowercase
  • Remove HTML tags
  • Remove punctuation & spaces
  • Remove stopwords (keep: not, no, never)
  • Lemmatize tokens
      │
      ▼
[ FastText Vectorization ]
  • Each token → 300-dim word vector
  • Mean pooling → single 300-dim sentence vector
      │
      ▼
[ Keras Neural Network ]
  • Input: (1, 300) vector
  • Output: confidence score [0.0 – 1.0]
      │
      ▼
[ Result ]
  • score ≥ 0.5 → ✅ Positive
  • score < 0.5  → ❌ Negative
```

## 🚀 Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/NLP-Sentiment-Analyzer.git
cd NLP-Sentiment-Analyzer
```

### 2. Create a Virtual Environment (Recommended)

```bash
python -m venv myenv
# Windows
myenv\Scripts\activate
# Linux/Mac
source myenv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r req.txt
python -m spacy download en_core_web_sm
```

### 4. 🏋️ Train the Models (Auto-Generate Model Files)

> ⚠️ The trained model files (`ReviewAnalyiser.h5` and `embeddingModel.bin`) are **not included** in the repository due to their large size.
> Run the training notebook to auto-generate them:

```bash
jupyter notebook app.ipynb
```

Run all cells in `app.ipynb`. This will:
- Preprocess the IMDB dataset
- Train the **FastText** embedding model → saves `embeddingModel.bin`
- Train the **Keras** deep learning classifier → saves `ReviewAnalyiser.h5`

### 5. 🌐 Launch the Web App

```bash
streamlit run web.py
```

Open your browser at **`http://localhost:8501`**

- **Source**: [IMDB Movie Reviews Dataset](https://ai.stanford.edu/~amaas/data/sentiment/)
- **Size**: 50,000 labeled movie reviews
- **Labels**: Binary — `positive` / `negative`
- **File**: `IMDB Dataset.csv`

---

## 🧪 Key Design Decisions

- **Mean Pooling** instead of padding/truncating — produces a fixed-size representation regardless of review length
- **Negation-aware stopword removal** — `not`, `no`, `never` are preserved to avoid flipping sentiment meaning
- **Lemmatization** — reduces vocabulary size and improves generalization
